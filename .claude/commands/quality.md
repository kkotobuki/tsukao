---
description: 静的コードゲート（fix→typecheck→deadcode→差分テスト）を順番に通す
---

実装の仕上げに、ローカルで **静的コードゲート** を一気に通してください（CLAUDE.md 共通ルール 7.1）。

## 目的（なぜこれが要るか）

コマンドを「順番に・回し忘れずに」通すための入口。順序を1コマンドに固定し、PR後に CI で気づく手戻りを防ぐ。
UI の破綻（ダークモード・スクロール・文字切れ）は**このゲートでは検出できない** → 通過後に `app-check` skill（7.2）へ続ける。

## 手順

1. **このプロジェクトの品質ゲートコマンドを特定する**
   - `CLAUDE.md` の「このリポジトリ固有のルール > 品質ゲート」を読む。具体コマンドはそこに書いてある。
   - 書かれていなければ、`package.json` の scripts（または対応するタスクランナー設定）から該当コマンドを推定し、**実行前にユーザーへ確認**する。
2. **次の順序で実行する**（順序厳守。書き換える工程を先に、検証を後に）:
   ```
   1. lint / format (autofix)   ← ファイルを書き換える。最初に確定
   2. typecheck                 ← 確定したコードに型検証
   3. deadcode 検出
   4. 差分テスト（変更関連のみ）  ← 最も重い検証を最後に
   ```
3. **1工程でも失敗したら、そこで止めて報告する**（後続は流さない）。失敗内容と該当箇所を提示する。
4. 全工程パスしたら、その旨を報告し、**続けて UI/目視ゲート（`app-check`）に進むか**をユーザーに確認する。

## 注意

- 具体コマンドを**この雛形に直書きしない**（テンプレは汎用ベース）。常に固有ルール / scripts から取得する。
- autofix でファイルが書き換わった場合は、何を変更したかを要約して伝える（黙って書き換えない）。
- 確認ダイアログを消したい場合は、対象コマンドを `.claude/settings.json` の permissions allowlist に追加する（`update-config` skill / `/fewer-permission-prompts`）。
