`/Users/jemka/Desktop/productivity improvement/scripts/measure_ai_usage.py` を `--out` 付きで実行し、生成されたレポートに診断と次アクションを追記する。

## Step 1: 事実データを自動生成

```bash
cd "/Users/jemka/Desktop/productivity improvement"
TODAY=$(date +%Y-%m-%d)
python3 scripts/measure_ai_usage.py --out "reports/${TODAY}.md"
```

スクリプトが以下を全て自動生成する:
- コアKPI（前30日比較あり、サンプル少なら自動 fallback）
- プロジェクト別表（前30日比較あり）
- モデル別利用比率 / トークン内訳
- グローバル整備度 / プロジェクト別整備度
- 末尾に「7. 診断・示唆」「8. 次アクション」のプレースホルダを残す

同日に複数回実行された場合は上書き（事実データなので問題なし）。

## Step 2: 情報源を読む

以下を順に読んで、診断・次アクションの根拠とする:

1. **`reports/${TODAY}.md`** — 生成された事実データ。**特に「直近30日 / 前30日 / 差分」列に注目**
2. **`reports/journal.md`** — 月内の出来事メモ。**同月のエントリは特に重要**（数値変化の原因が書かれている可能性大）
3. **`reports/` 配下の直近の `YYYY-MM-DD.md`**（今回生成したものを除く） — トレンドの読み解き補助

## Step 3: 診断・次アクションを追記

生成された `reports/${TODAY}.md` の末尾にあるプレースホルダ:

```
## 7. 診断・示唆

_（Claude との対話で追記。...）_

## 8. 次アクション

_（Claude との対話で追記。...）_
```

を **Edit ツールで実内容に置き換える**（同じファイル内で完結）。

### 「7. 診断・示唆」に書くこと

- 数値変化（差分列）から読み取れる主要な傾向（3-5項目）
- journal.md の今月エントリと数値変化の対応関係（例: 「hooks 登録 → セッション完遂率 +X.Xpt」）
- 異常値・注目点（コスト急増した特定プロジェクト、整備度ゼロのプロジェクト等）
- 不明な点があれば「要ヒアリング」として明示し、推測で埋めない
- 前30日比較が `fallback` モード（サンプル少）の場合、その旨を明記して比較を保留

### 「8. 次アクション」に書くこと

- 診断から導かれる整備候補に `[P0]` `[P1]` `[P2]` の優先度ラベル
- 出典がある場合は明記（journal の特定エントリ、スキル化候補レポート、等）
- 既に進行中があれば「継続: ...」と明示

## Step 4: ユーザーへサマリ表示

レポート完成後、以下の形式で表示:

```
✅ 月次計測完了: reports/<YYYY-MM-DD>.md

📊 コアKPI（直近30日 / 前30日比）
- CLI自律倍率: <値>倍 (<差分>)
- キャッシュヒット率: <値>% (<pt差>)
- 月間推定コスト: $<値> (<差分>)
- セッション完遂率: <値>% (<pt差>)

🎯 主な気づき:
- (診断セクションの top 2-3 を箇条書きで)

📌 次アクション (P0/P1 のみ):
- ...
```

## 注意事項

- レポートのファイル名は **`YYYY-MM-DD.md`**（ハイフン区切り日付）。旧形式 `AI活用率_計測レポート_*.md` は使わない
- journal.md に当月エントリが無い場合は、診断が薄くなる旨をユーザーに通知し、追記をお勧めする
- レポート生成完了したら、ユーザーが追加で何か聞かない限り、それ以上の説明や提案はせず終了
