description: 保存されたチェックポイントから作業を再開する
argument-hint: <tag-name>（省略時は最新を使用）
---

グローバルチェックポイント領域から保存されたチェックポイントを読み込み、作業を再開してください。

## 引数

- `$ARGUMENTS` にユーザーが指定したタグや日付が入る
- 未指定の場合は **最も新しいチェックポイント** を使用（`_latest.md` > 最新の手動checkpoint の順）

## 保存場所の特定

現在の作業ディレクトリ（`pwd`）をエンコード:
- `[^a-zA-Z0-9._]` を全て `-` に置換
- 結果を `~/.claude/checkpoints/<エンコード>/` の `<エンコード>` 部分とする

例: `/Users/jemka/Desktop/EVeM/survey/survey_monorepo`
→ `~/.claude/checkpoints/-Users-jemka-Desktop-EVeM-survey-survey_monorepo/`

## 動作

### Step 1: 利用可能なチェックポイントを確認

```bash
CHECKPOINTS_DIR="$HOME/.claude/checkpoints/<エンコード>"
ls -t "$CHECKPOINTS_DIR"/*.md 2>/dev/null
```

ファイルが無ければ「このプロジェクトのチェックポイントが見つかりません」と伝えて終了。

### Step 2: 対象のチェックポイントを選定

- 引数あり: ファイル名に引数文字列を含むものを探す（部分一致OK）
  - 該当が複数なら一覧を表示してユーザーに選択させる
  - 該当が0件なら全件を一覧表示
- 引数なし:
  - `_latest.md` が存在し、frontmatter の `created` が24時間以内なら使用
  - そうでなければ最新の手動checkpointを使用

### Step 3: 期限チェック

frontmatter の `expires` を確認。
- 期限切れの場合: ユーザーに「期限切れですが復元しますか？」と確認

### Step 4: 内容を読み込んでサマリ表示

選んだチェックポイントを `Read` ツールで読み、以下の形でユーザーに提示:

```
📂 チェックポイントを復元しました

**ファイル**: ~/.claude/checkpoints/<エンコード>/<filename>
**保存日時**: <created>
**タグ**: <tag>
**ブランチ**: <git_branch>

## 前回やっていたこと
<やっていたこと セクションの内容>

## 次にやることリスト
<次にやること セクションの内容>

## 関連ファイル
<コンテキスト セクションのファイル一覧>

---

このまま続きから着手してよろしいですか？
```

### Step 5: ブランチ確認

frontmatter の `git_branch` と現在のブランチを比較:
- 異なっていれば「現在ブランチが <現在> ですが、チェックポイントは <保存時> です。切り替えますか？」と確認

### Step 6: 着手

ユーザーが「OK」「進めて」「お願い」等の応答を返したら、「次にやること」の最初のTODOに着手する。

## 重要

- チェックポイントを読んだら、そこに書かれている内容を **このセッションの作業前提として扱う**
- 「次にやること」が明確なら確認せず即着手しても良い
- ただし **破壊的操作（git push、デプロイ等）の前は必ず一度確認** する
- グローバル領域（`~/.claude/checkpoints/`）のみ参照。プロジェクト内 `.claude/checkpoints/` は参照しない
