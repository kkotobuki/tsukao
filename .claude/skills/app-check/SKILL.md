# アプリ作成時チェックリスト（app-check）

静的ゲート（型 / lint / deadcode / 差分テスト）を全部通しても壊れている典型 ── **ダークモードの抜け・スクロール不能・文字切れ** ── を拾うための、**目視中心の最終確認ゲート**。

本体のチェック項目は [`checklist.md`](./checklist.md) にある。このスキルはその「回し方」を定義する。

---

## 設計思想（重要）

このチェックリストは **全項目を載せる（網羅）** が、**全項目を毎回ブロックしない（取捨）**。
固いゲートにして「全項目パスしないと終われない」運用にすると、小さな修正でも全チェックを強制されて形骸化する。だから各項目を**タグで仕分け**して、状況に応じて取捨する。

### タグの意味

- **[自動]** … Playwright 等で機械的に確認できる。`webapp-testing` skill で回す。
- **[目視]** … 人が画面を見て判断するしかない。スクショを撮って一覧化し、人に委ねる。
- **[必須]** … UIがあるアプリなら原則すべて確認すべき。
- **[該当時]** … その機能がある時だけ確認（例：フォームが無いアプリにバリデーション確認は不要、ダークモード非対応なら該当せず）。

---

## 回し方

### 1. 適用範囲を絞る
今回の変更で触れた画面・機能だけを対象にする。全画面を毎回見る必要はない（[該当時] を落とす）。

### 2. [自動] 項目を機械で回す
`webapp-testing` skill（Playwright）と `run` skill を使い、以下を自動実行する：

1. `run` skill でアプリを起動
2. 対象画面を開き、`checklist.md` の **[自動]** 項目を検証
   - ダークモード：テーマ切替後に `body` 等の computed 背景色・文字色が変わっているか
   - スクロール：`scrollHeight > clientHeight` なのにスクロール不能な要素、意図しない横スクロール（`scrollWidth > clientWidth`）
   - 文字切れ：`text-overflow: ellipsis` で隠れたテキスト、要素からの溢れ
   - console エラー・警告、失敗したネットワークリクエストの収集
   - レスポンシブ：複数 viewport でスクショ＋横スクロール検出
   - アクセシビリティ：可能なら axe-core でコントラスト・ラベル不足
3. 各画面のスクショを撮る

### 3. [目視] 項目をチェックリストとして提示する
自動で判定しきれない項目（配色・余白・トンマナ、空/ローディング/エラー状態の見た目など）は、**撮ったスクショ＋未チェックの markdown チェックリスト**を出力し、人の判断に委ねる。Claude が勝手に「OK」にしない。

### 4. レポートにまとめる
- [自動] 検出結果（パス/要対応）
- 各画面のスクショ
- [目視] 必須項目のチェックリスト（未チェック状態で提示）

---

## 静的ゲートとの関係

| ゲート | 内容 | 入口 |
|---|---|---|
| 静的コードゲート | 型 / lint / deadcode / 差分テスト | `/quality`（スライドの構成） |
| **UI/目視ゲート** | **本スキル（ダークモード・スクロール・文字切れ 等）** | **`app-check`** |

この2本立てで「コードは通るが画面が壊れている」を塞ぐ。順序は **静的ゲート → app-check**。

---

## 注意

- **[目視] 項目を Claude が独断で「確認済み」にしない。** 必ず人に提示する。
- アプリの種類で [自動] の手段は変わる（Web=Playwright、モバイル/ネイティブ=スクショ運用中心）。[自動] が使えない環境では、その項目も [目視] として扱う。
- このリストは「触った範囲」に絞って使う。全項目の一律強制はしない。
