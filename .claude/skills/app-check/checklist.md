# アプリ作成時チェックリスト（完全版）

UIが存在するアプリを作成・修正したら確認する項目。タグの意味は [`SKILL.md`](./SKILL.md) を参照。

- **[自動]** 機械で確認できる / **[目視]** 人が見て判断
- **[必須]** UIがあれば原則確認 / **[該当時]** その機能がある時だけ

> 使い方：今回触れた画面・機能に絞ってチェックする。[該当時] で機能が無いものは飛ばす。全項目の一律強制はしない。

---

## A. UI・見た目（一番漏れる）

- [ ] [自動][必須] ダーク/ライトモードが **全画面** で通っている（背景・文字・境界線・アイコンまで。一部だけ白浮き/黒浮きしていないか）
- [ ] [自動][必須] スクロールできる（縦スクロールが必要な画面で実際に最後まで到達できる）
- [ ] [自動][必須] 意図しない横スクロールが出ていない（`scrollWidth > clientWidth`）
- [ ] [自動][必須] 文字切れ・はみ出しが無い（`…` で重要情報が隠れていないか、ボタン内テキストの溢れ）
- [ ] [自動][必須] レスポンシブ崩れが無い（SP / タブレット / PC の各幅でレイアウトが破綻しない）
- [ ] [目視][該当時] 空状態（データ0件）の表示が用意されている（真っ白・崩れではなく、メッセージや導線）
- [ ] [目視][該当時] ローディング状態の表示がある（スピナー/スケルトン。無言の固まりに見えない）
- [ ] [目視][該当時] エラー状態（取得失敗・通信エラー）の表示がある（白画面・無反応ではない）
- [ ] [目視][必須] 長文・極端な入力でレイアウトが崩れない（長い名前・大量データ・0件）
- [ ] [自動][必須] 画像切れ・アイコン抜けが無い（404 画像、読み込み失敗）
- [ ] [目視][必須] フォント・配色・余白が意図通り（指定フォントが当たっている、トンマナが揃っている）

## B. 動作・機能

- [ ] [目視][該当時] フォームのバリデーションが効く（必須 / 形式 / 文字数上限 / 異常値）
- [ ] [自動][必須] エラーで画面が落ちず、メッセージが出る（白画面クラッシュしない）
- [ ] [目視][必須] リロード・ブラウザバックで壊れない（状態が飛ぶ/二重表示しない）
- [ ] [目視][該当時] 二重送信防止が効く（連打でボタンが多重実行されない）
- [ ] [目視][必須] 主要導線（ハッピーパス）が一通り通る（最初から最後まで操作できる）
- [ ] [目視][該当時] 権限・ログイン状態による出し分けが正しい（未ログイン/ログイン済み）

## C. 静的ゲート（コード品質）

> ここは `/quality` に集約。app-check の前に通しておく。

- [ ] [自動][必須] 型チェック（typecheck）が通る
- [ ] [自動][必須] lint / format（autofix 含む）が通る
- [ ] [自動][必須] デッドコード（knip 等）の検出がクリーン
- [ ] [自動][必須] 変更に関連するテスト（差分テスト）が通る
- [ ] [自動][必須] console のエラー・警告がゼロ
- [ ] [自動][必須] 失敗したネットワークリクエストが無い（404 / 500 / CORS）

## D. アクセシビリティ

- [ ] [自動][必須] コントラスト比が基準を満たす（特にダークモードで見えにくくないか / axe-core）
- [ ] [目視][必須] キーボード操作・フォーカスが機能する（Tab 移動、フォーカスリングが見える）
- [ ] [自動][該当時] 画像 alt・フォームラベル・ボタンの aria が付いている

## E. 非機能

- [ ] [目視][必須] 初期表示が極端に遅くない（重い同期処理・巨大バンドルで固まらない）
- [ ] [目視][該当時] 秘密情報（API キー・トークン）が画面・ソース・ログに露出していない
- [ ] [目視][該当時] 認証・認可が必要な画面が保護されている

---

## 補足：このリストの育て方

- 実際に「やり忘れて後から気づいた」項目が出たら、ここに1行足す。リスト自体を運用で育てる。
- 逆に「自分のプロダクトでは絶対不要」な項目は、プロジェクト側でコメントアウトして良い（テンプレ本体は消さない）。
