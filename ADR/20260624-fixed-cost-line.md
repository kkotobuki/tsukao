# 消費の「固定費／自由」分解と固定費線

**ステータス**: Accepted（2026-06-24）

**読者**: ツカオ（Life GPS）の実装者（本人）。**目的**: 収入・支出グラフの「消費」一本線を「固定費（必須）」と「自由に使えるお金」に分解して見せる判断と、必須／自由の線引き（比率 r）の根拠を記録する。

# Motivation

## 問題

現状の UI は、収入・支出グラフに「消費」を **合計一本** でしか出していない。そのため本人から見て次が分からない：

- 消費の中身が必要な分なのか自由に使える分なのか分からない（「消費ってどんなものか分かりたい」）
- 「今、経験にお金を使っていい」というツカオの中核体験に必要な、**自由に動かせるお金の量**が読み取れない

入力は「毎月の支出」を1つの数字で取っているだけで、費目内訳は持っていない。一方で計算エンジンは住居費（家賃／購入後ローン）を区別して持っており、固定費の一部は既存入力から確定できる。

## 目的

1. 消費線を「必須（固定費）」と「自由」に分解し、両者の境目を図に出す
2. 入力負荷を増やさない（費目の追加入力はしない）
3. 住居費を二重計上しない
4. 平均一本線・自己参照的充足の思想と整合させる（家計簿化＝費目分解はしない）

# Decision

## D1. 固定費線を1本、収入・支出グラフに足す

「消費」線の下に「固定費」線を引く。両線の差が「いま自由に使えているお金」になる。費目の追加入力欄は設けない。

## D2. 固定費の算出式（住居費＋必須生活費）

各年で：

```
固定費 = 住居費 ＋ (基本消費 − 住居費) × 必須比率 r
自由   = 基本消費 − 固定費
```

住居費は二重計上を避けるため住居プランで切り替える（`fixedCostForYear` in `simulate.ts`）：

| 住居プラン | 住居費分（消費に含まれる家賃） |
|---|---|
| 賃貸 | currentRentYen |
| 購入（購入前） | currentRentYen |
| 購入（購入後） | 0（家賃は消費から除かれ、ローンはイベント側で計上） |
| 持ち家 | 0（家賃なし） |

退職後は基本消費＝最低生活費（全額が必須）とみなし、固定費線は消費線に重なる。

## D3. 必須／自由の費目分類と比率 r

非住居の基本消費を、家計調査の費目構成でこう分類する：

| 必須（固定費に入れる） | 自由（裁量＝経験に使えるお金） |
|---|---|
| 食料（自炊）・光熱水道・保健医療・交通通信・家具家事用品 | 外食・教養娯楽・被服・交際費・その他 |

**r = 0.55**（`snapshot.necessityRatioOfNonHousing`）。家計調査2024 単身世帯の費目別から較正（2026-06-25）。

較正の計算（月・円、住居39,620と教育0を除いた非住居計136,536で割る）:

| 費目 | 月額 | 区分 |
|---|---|---|
| 食料 | 43,407 | 必須（自炊）／自由（外食） |
| 光熱・水道 | 9,004 | 必須 |
| 家具・家事用品 | 4,649 | 必須 |
| 被服及び履物 | 8,102 | 自由 |
| 保健医療 | 8,276 | 必須 |
| 交通・通信 | 19,681 | 必須 |
| 教養娯楽 | 24,961 | 自由 |
| その他（交際費等） | 18,456 | 自由 |

- 食料を全部必須とすると r = 85,017 / 136,536 ≈ **0.62**
- 外食（食料の約3割）を自由へ移すと r = 71,995 / 136,536 ≈ **0.53**
- 範囲 0.53〜0.62 の中間寄りで **0.55** を採用（外食の一部を自由に寄せた）。出所: 総務省 家計調査（家計収支編）2024 単身世帯。
- 残る不確かさ: 外食・交際費の個別内訳が公表表から取れていない（約3割は推定）。内訳が取れたら再較正（TODO）。

# Rationale

- **なぜ費目分解（家計簿）にしないか**: 入力負荷が上がり、平均一本線・自己参照的充足（細かく管理させない）思想と逆方向。固定／自由の2層で十分に「消費の中身」を返せる。
- **なぜ計算で出すか（入力欄を足さないか）**: 住居費は既存入力から確定でき、残りは統計比率で推定できる。本人が決めた線でなく統計の押し付けに見えないよう、生活費に比例（毎月支出 × 比率）させ自分ごと性を保つ。
- **なぜ住居費を分けるか**: 家賃／ローンは確定した固定費であり、家計調査の住居費目（持ち家の帰属家賃を含まない等）と二重計上すると過小／過大になるため、住居プランから直接取る。
