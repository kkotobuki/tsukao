## 0. プロダクト一行サマリ

「自分専用の将来シミュレータ」。漠然とした**お金の将来不安**を、**具体的な見通し**と**"自分で動かせる感覚（コントロール感）"**に変えるツール。年齢・年収・資産・支出から「今の現在地」と「ありうる未来」を**現在価値（今日の円）**で描き、最終的に問うのは **"その未来で自分は満足か"**（数字は手段、満足が目的）。他人とは比較しない自己参照。

**ペルソナ**: 開発者本人。お金の将来不安が強い。結婚・子供・住宅への欲求は薄め（「やりたい」より「**もし選んだらどうなるか確認したい**」）。他人より下かでなく「**自分が満足できるか**」を知りたい。エンジニアなので数字の根拠を理解し、自分でいじりたい。

**真の目的**: 不安を煽らず、(1) 見通しを具体にする (2) レバーを動かして「**未来を自分で握れる**」コントロール感を与える (3) 「満足か」を自分に問い感じてもらう。

## 1. v1 スコープ

### やる
- 入力（現在地）→ 将来の貯蓄推移を**平均一本線（決定論）**で計算 → 表示
- **what-if サンドボックス**（入力をいじると未来が引き直される）
- 退職時資産を「**今の生活費なら◯年分**」に体感変換して提示
- やわらかな資産推移の可視化（**スコア・勝敗・順位なし**。ゲーム要素は持たない／双六メタファーは廃止）
- ライフイベント（結婚・子供・住宅 等）の任意 ON/OFF で費用加算
- （保存レイヤ）主観反応ログ・自己モデル・シナリオを Notion 保存
- （AI相談レイヤ）相棒型のAI相談

### やらない（v1）
- 確率分岐／モンテカルロ（平均一本線のみ。テールリスクは構造上出さない）
- 物価（インフレ）計算（すべて現在価値）
- 他人との比較・順位・percentile・スコア
- Supabase／独自認証（個人アプリの範疇を超えたら移行）
- 転職リスクのモデル化（無視）

### v1 を「保存なし・中継なし」で出せる
読む＋計算＋what-if は端末内で完結（前提値はビルド時スナップショット同梱）。**保存とAI相談を足す段で初めて中継（サーバ）が要る**（→ §6）。

## 1.5 実装セッションへの申し送り（2026-06-19・要反映）

ユーザー確定の追加決定。実装ADR追補(2026-06-19)の一部を**改訂**する。**実装側へ反映が必要**:

1. **消費は年齢カーブで増減**（決定F「消費一定」を改訂）。`消費増加率`を base に掛ける。**率は導出済み（§3.3）**: 単身は現役ほぼ横ばい(+0.3%/年) → 60歳で約 −14% → 以降 −0.6%/年。※現役はほぼ一定なので決定Fの現役期の誤差は小さいが、**退職後の消費減**を入れると老後の足り具合が改善する（より正確）。
2. **住宅＝家賃置換**。`住居プラン`（ずっと賃貸／◯歳で購入／ずっと持ち家）に統合（旧「住居形態」＋住宅購入イベント）。◯歳で購入なら購入年から base の家賃を消し維持費＋ローンへ置換。ずっと持ち家は置換不要。
3. **オプション3類型で二重計上回避**（状態変化/特定上乗せ/独立・§3.3）。base に平均が入る費目（趣味娯楽・理美容・自動車・医療）はオプションで重ねない。
4. **趣味娯楽・健康美容は「特定追加分のみ」**。汎用"拡大"でなく「特定の趣味」「整形/特定の美容」を本人が枠追加して入力するカスタム形に。
5. **退職後は「N年レバー」方式**（決定G の寿命まで年次ループ・`depletionAge` を改訂）。**死ぬ年齢は入力しない**。年次ループは退職年齢で止め、退職後は閉じた式に: `年間自由支出 = 年金(種類別・§3.7) + パート + (退職時貯蓄 ÷ N) − 最低生活費`。N（貯蓄を使う年数）は自由入力レバー。年金は種類別（国民≈5.9万/厚生≈15.1万・月）、退職後最低生活費≈14.9万/月。

## 3. データモデルと既定値（すべて現在価値・実質）

### 3.1 ユーザー入力（基本情報＝現在地）
今あなたが実際に払っている／得ている**実額**を入力する（平均値ではない）。

| パラメータ | 型 | 例 |
|---|---|---|
| 現在年齢 | int（歳） | 26 |
| 現在の年収（額面） | int（円/年） | 4,000,000 |
| 現在の金融資産 | int（円） | 入力 |
| 現在の毎月の支出（総額） | int（円/月） | 入力。固定費＋生活費＋**今かかっているオプション費用すべて**を含む総支出 |
| 退職年齢 | int（歳） | 65 |
| 年金の種類 | enum | `国民年金のみ`（自営/フリー）/ `厚生年金`（会社員）。種類で受給額が決まる（→ §3.7） |
| 性別 | enum（任意） | `男性` / `女性` / 未回答。予測寿命（平均余命）の参照にのみ使い、未回答なら男女平均（ADR: 20260703-spend-down-projection） |
| ~~貯蓄を使う年数 N~~ | — | **入力レバー廃止**（ADR: 20260703）。残り年数＝退職〜予測寿命で自動化。死ぬ年齢は入力しない（→ §4.3） |
| 世帯パターン（現在の婚姻状況） | enum | `単身` / `夫婦のみ`（将来の結婚はイベントで足す） |
| 住居プラン | 選択 | `ずっと賃貸` / `◯歳で購入`（家賃→ローン＋維持に置換＋頭金）/ `ずっと持ち家`。賃貸/購入を**1つの選択軸**に統合（旧「住居形態」＋住宅購入イベントを統合。`SimulationInput.housingPlan`） |
| 退職後パート | bool＋年額 | false（→ §3.6） |
| 投資する | bool | false |
| 消費の基準（consumptionBasis） | enum | `fromSavings`（手取り−毎月貯蓄×12・既定）/ `average`（単身平均）/ `explicit`（直接指定）。→ §3.3 |

### 3.2 収入パラメータ（実質・年齢昇給率）
賃金センサスの年齢カーブから導出（実質＝物価を含まない年齢プレミアム）。**現在の年収（額面）を起点に率を掛けて伸ばす**。※実装の昇給カーブDB／スナップショットは**令和6年の絶対年収**（所定内×12＋賞与）を使用し、コアはこの絶対カーブを `年収(age)/年収(現在年齢)` で正規化して参照する。下表の率は参考の近似値。

| 年齢帯 | 年率昇給率 |
|---|---|
| 26–30 | +2.3% |
| 31–35 | +1.8% |
| 36–40 | +1.4% |
| 41–45 | +0.7% |
| 46–50 | +0.6% |
| 51–55 | +0.4% |
| 56–60 | ≒0%（横ばい） |
| 60超（再雇用） | −3.0〜−3.6% |

- 簡略実装用の代表値: **+1.2%/年**（20代後半〜50代）
- 賃金ピーク = 55–59歳
- `実質ベースアップ率`（物価超の実質賃上げ）= 既定 **0%**（任意で上げられる）
- ※春闘賃上げ率はそのまま掛けない（定昇＝年齢昇給率／物価＝不使用／実質ベアに分解済み。二重計上回避）

### 3.3 消費（年齢カーブで増減 ＋ オプション3類型）

ベース消費は**年齢で増減する**（`消費増加率`の年齢カーブ＝中年でピーク・退職後は減）。物価は無視（現在価値）。税・社保は消費に含まない。

- **基準の置き方 `consumptionBasis`**（今の消費水準をどこから取るか）: `fromSavings`（手取り−毎月貯蓄×12＝本人の実態・**既定**）／ `average`（家計調査の単身平均）／ `explicit`（直接指定）
- baseには**平均的な趣味娯楽・理美容・自動車・医療が既に含まれる**。よってオプションは「平均を超える分／状態変化」だけを足す（下の3類型）。

**オプション3類型（二重計上を防ぐ）**

| 類型 | 例 | 扱い |
|---|---|---|
| **状態変化** | 結婚（増分）・子供（増分・教育費込み）・住宅購入（**賃貸なら家賃を置換**／持ち家は適用外） | 世帯・住居の構造が変わる |
| **特定の上乗せ（カスタム）** | 特定の趣味（旅行・ゴルフ等）・整形/特定の美容健康・特定の保険 | baseの平均を超える**自分固有の追加分**を本人入力 |
| **独立** | 大病（スポット）・介護施設・退職後パート（収入） | baseと無関係 |

- **住宅（住居プランに依存・§3.1）**: `◯歳で購入` を選ぶと購入年から base の家賃を消し維持費＋ローンに**置換**。`ずっと持ち家`なら家賃は元々baseに無い（家計調査は持ち家の帰属家賃を計上しない）ので置換不要。
- **結婚増分** = `夫婦のみ(293,511) − 単身(173,042) ≒ +120,469円/月`（単身baseに加算）。
- **`消費増加率`（年齢カーブ・単身・実質）＝導出済み**（家計調査2024 単身世帯）。物価上昇とは別物（物価は無視で確定）。単身は**現役ほぼ横ばい・退職後に減少**が特徴:

| 年齢帯 | 月消費（2024単身） | 消費増加率（年率・実質） |
|---|---|---|
| 〜34歳 | 176,160円 | （起点） |
| 35〜59歳 | 184,749円 | 現役: 約 **+0.3%/年**（ほぼ横ばい） |
| 60歳以上 | 159,249円 | 退職期: 約 **−1.0%/年**（または60歳で約 −14% の段差） |
| 65歳以上 | 154,601円 | 約 **−0.6%/年**（緩やかに減） |

  出所: 家計調査2024 単身世帯（年齢別月消費）。バンドが粗い（〜34/35〜59/60〜）ため、**現役は実質ほぼ一定／効くのは退職後の減少**。簡略実装は「現役一定 → 60歳で −14% → 以降微減」でよい。

### 3.6 ライフイベント費用（任意トグル）
Notion DB「ライフイベント別 年額コスト（前提値）」に約20行・18プリセット。各行 = `年額の目安(万円)` / `初期費用(万円)` / `計算区分`。**ONにした項目の年額（＋初期費用）を加算**。代表例:

| イベント | 年額 | 初期費用 | 計算区分 |
|---|---|---|---|

| 介護施設入居 | 180万 | 47.2万 | 毎年支出 |
| 大病・長期療養 | 0（毎年でない） | 150万 | **一回(スポット)** |
| 退職 | — | — | **収入転換**（年収→年金） |
| 退職後パート（部分収入） | （収入・年額） | — | **収入転換**（退職後にプラス。例: 年100万・要確認/手入力） |

| 海外居住・住み替え 等 | 手入力 | — | 手入力 |

- 結婚は世帯パターンで扱う（DBの「結婚」行は初期費用＝挙式等。年額≒0）

### 3.7 年金（種類別）と退職後の最低生活費（現在価値・月額）

**年金（厚労省 令和6年度 概況）** — 種類で受給額が違う。ユーザーが種類を選ぶ:

| 種類 | 平均月額 | 備考 |
|---|---|---|

## 4. 計算仕様（コアロジック）

すべて現在価値（物価なし）。年次ループは t=0（現在年齢）→ **退職年齢まで**（退職時貯蓄を積む）。退職後は **§4.3 の使い切りビュー**（予測寿命をゴールに、使い切りペースを逆算）で扱う（死ぬ年齢は入力しない）。

### 4.1 現在地（t=0）
```
手取り年収(0)   = 額面年収 → 税・社保控除（§3.4）
月の余り        = 手取り年収(0)/12 − 現在の毎月の支出
現在地（資産）  = 現在の金融資産
```

### 4.2 将来（t = 1 … 退職年齢まで、年次）— 退職時貯蓄を積む
```
# 収入（現役のみ。退職後は §4.3 の閉じた式へ）
額面年収(t) = 額面年収(t-1) × (1 + 年齢昇給率[年齢] + 実質ベースアップ率)
手取り(t)  = 額面年収(t) → 税・社保控除

# 支出（§3.3。収入転換・投資は上の収入側に適用済み）
基礎消費(t) = consumptionBasis の現在水準 × 年齢カーブ係数(年齢)        # 消費増加率の年齢カーブ（§3.3・導出済み）
消費(t)     = 基礎消費(t)
            − 家賃（住宅購入ON かつ 賃貸 の年以降。家賃置換）
            + Σ 状態変化・特定上乗せイベントの年額（終了年内・決定E）+ スポット + 結婚増分（結婚後）
# 年齢カーブは中年ピーク→退職後減。物価は乗せない（現在価値）

# 資産更新（モード非依存）
資産(t) = 資産(t-1) × (1 + 実質運用利回り)            # 投資ON時のみ >0
        + 手取り(t)
        − 消費(t)
        − 初期費用（消費系イベント開始年のみ）
# このループは退職年齢で終了。最終年の資産＝退職時貯蓄 → §4.3 へ
```

### 4.3 退職後の出力（使い切りビュー。死ぬ年齢は入力しない。ADR: 20260703-spend-down-projection）
```
予測寿命       = 現在年齢 + 平均余命(現在年齢・性別) + 寿命の伸び率 × (退職を迎える年 − 2024)
  ※令和6年簡易生命表 ＋ 社人研 将来推計（死亡中位）。性別未回答は男女平均。入力はさせない
年金年額       = 年金月額(種類別・§3.7) × 12
パート年額     = 退職後パート（任意）
最低生活費年額 = 退職後の最低生活費(§3.7・約14.9万/月) × 12
退職時貯蓄     = §4.2 ループの最終年の資産

使い切り年額 = 「予測寿命で資産がちょうど0になる年間支出」
  ※年金＋パート・退職後イベント・運用益込みの退職後シミュレーションを二分探索で解く
  ≥ 最低生活費 → 使い切りモード: 「寿命までに 月◯円 使えます。使い切れますか？」
                 グラフは使い切りペースで下りながら予測寿命でゼロ着地する線
  < 最低生活費 → 不足モード: 最低生活費で取り崩し、尽きる年で打ち切る線＋打ち手
                 （働く期間↑ / 支出↓ / 貯蓄↑ を what-if）

年間自由支出 = 年金年額 + パート年額 + (退職時貯蓄 ÷ 残り年数) − 最低生活費年額
  ※残り年数 = 予測寿命 − 退職年齢。「今の許可」ソルバ（§5.6）の判定に使う閉じた式
  ※介護・大病等を退職後に置いた分は、退職時貯蓄から先に差し引く
```
- 体感変換は「使い切り年額 ÷ 今の生活費」＝**今の生活費の◯倍**（月額換算も併記）。
- 死ぬ年齢は入力させない・予測寿命は統計の平均（平均一本線と同じ「平均どおり進めば」の基準線）。
- 中核アウトプットは金額でなく「**満足か**」。提示後に主観反応（安心/しんどい）を記録（§5.3）。

### 4.4 what-if
- 入力（年収・支出・退職年齢・世帯パターン・ONイベント・投資）を変えると**即再計算**して線を引き直す
- 一度きりの静的表示ではなく、その場で「もしこうしたら」を試せる操作を一級市民に

## 5. UX / 画面

### 5.1 UI原則（ペルソナ＝不安の強い本人に効かせる）
- **不安を煽らない**: 赤字や「詰み」を主役にしない。不足時も「働く期間↑ / 支出↓ / 投資 でこう変わる」と**打ち手を必ず一緒に見せる**。警告色は赤でなく穏やかな琥珀。
- **コントロール感が主役**: レバー（退職年齢・N・年収・貯蓄・投資・イベント）を動かすと**主役の数字が即変わる**。「未来を自分で握れる」体験が不安の解毒。
- **比較しない・断定しない**: 順位/percentile/スコア/勝敗なし。「あなたは大丈夫/ダメ」でなく「こうすると こう変わる」。
- **抽象を体感に**: 「退職後 年 △△円 自由に使える」「今の生活費なら ◯年分」で見せる。
- **根拠が追える**: 数字はタップで出所（家計調査・賃金センサス等）を出せる＝エンジニア的な安心。
- **感情を残す**: 結果に「どう感じた？」を軽く記録（満足が目的）。
- ゲーム要素・双六メタファーは持たない。

### 5.2 画面と流れ（ホーム → 計算 → 結果 → 振り返り → 提案）
- **状態**: 端末内 React state（`useReducer` で `SimulationInput` ＋ `Scenario`〔ONイベントの `PlacedEvent[]`〕）。`assumptions` は `snapshot.ts → mappers.ts`（`年金の種類`で `pensionAnnualYen` を厚生/国民に切替）。
- **計算は明示ボタン式**（即時再計算ではない）: 入力を整える → **[計算する]** → 「**計算中です**」のモーション（演出。実計算は同期で一瞬だが、結果に重みを持たせ一拍おく＝0.8〜1.5秒）→ 結果。
- **what-if** = 入力/オプションを変えて再度 [計算する]。

### 5.3 ホーム（入力）
**基本入力**（§3.1 必須）を上に、`SimulationInput` にマッピング:

| UI | コントロール | core フィールド |
|---|---|---|
| 現在年齢 | 数値 | `currentAge` |
| 年収（額面） | 数値（万円→×10000で円） | `grossAnnualIncomeYen` |
| 現在の貯蓄 | 数値 | `currentAssetsYen` |
| 毎月の貯蓄額 | 数値 | `monthlySavingsYen` |
| 退職年齢 | スライダー/数値 | `retirementAge` |
| 年金の種類 | セグメント（国民/厚生） | → `pensionAnnualYen` 切替 |

**「詳細を開く」エクスパンダ**（progressive disclosure。普段は畳む＝欲求薄めのペルソナに合う）。中で:
- **オプション（ライフイベント）の追加・ON/OFF**: `SNAPSHOT.events`（19件）＋カスタム追加。各行 トグル＋開始年齢＋count＋金額override。ONで `scenario.events` に `PlacedEvent` 追加・OFFで除去。結婚＝`semantic:'marriage'`、住宅＝住居プラン。
- 住居プラン（賃貸/持ち家/◯歳で購入）・消費の基準（既定 fromSavings）・退職後パート・投資（毎年いくら）・N（貯蓄を使う年数）

下部に **[計算する]** ボタン。

### 5.4 計算中
「計算中です」のローディングモーション（演出の一拍）→ 結果へ遷移。

### 5.5 結果
1. **年次グラフ（3系列）**: **資産額・手取り・消費額** を1年ごと（`YearProjection.assetsYen / netIncomeYen / consumptionYen`）。退職点をマーク。
2. **退職後サマリ**: 退職時の **残資産**、**年金額**（種類別）、退職後「**年 △△円 自由に使える**」（N年レバー・§4.3）。
3. **体感変換**: 「今の生活費なら **◯年分**」（`retirementYearsOfLivingCost`）。
- 不足が出ても赤で脅さず、§5.6 の打ち手を併記。
- ※ N年レバー出力はコアに未実装（現状 `depletionAge`）→ **申し送り5 で追加**。

### 5.6 振り返りと提案（このアプリの核＝"今を楽しむ許可"）
結果を見せた後、**次の問い**へ進める:
1. **振り返りの問い**:「思っていたより **多い？ / 少ない？**」（タップ）＝主観反応の入り口。
2. **多い（余裕あり）と感じた → "今をもう少し楽しむ"提案**:
   - 「実は ◯年分 の余裕。今の生活に 月 △△円 まで上乗せできる」と**許可**を出す
   - **新しく始めること** or **我慢してきたこと**をやるメリットを、選択肢として提示（旅行・趣味・学び 等。押し付けない）
   - ＝ AI相談ADR「**もう足りてる も返す**」の具体化。v1は**ルールベース**（余裕額に応じた提案文）でよい
3. **少ない（不安）と感じた → 打ち手**: 「働く期間↑ / 支出↓ / 投資」を1タップで試せる（脅さない・コントロール感）。
- 反応と気づきは（保存レイヤ §6 後に）主観反応ログ／自己モデルへ。

#### 5.6.1 「今 月△△円まで上乗せできる」の算出（実装・2026-06-24）
核アウトプット「今の生活に 月△△円 まで上乗せできる」は、退職後サマリの自由支出（§5.5）とは**別の数字**で、現役の今いくら増やしてよいかを直接解く（`solvePresentMonthlyHeadroomYen`）。
- **定義**: 現役期の毎月支出を Δ 増やしても **退職後の最低生活が守れる**（`annualFreeSpendingYen ≥ 0`）上限 Δ。`simulate` を再評価しながら二分探索で求める（モデルは変えない）。
- **閾値＝天井（案X・確定）**: SPEC本文「月△△円**まで**」に忠実に、退職後最低生活を守れる**上限ギリギリ**を提示する（buffer を引いた控えめ版＝案Y は採らない）。余裕<0（退職後が最低生活すら賄えない）なら 0＝上乗せ余地なし→打ち手側へ。
- **既知の性質**: 本モデルは「実質昇給しても生活水準を上げない（消費は現在価値で一定・§3.3）」前提のため余剰が大きく出やすく、上乗せ可能額も大きく出る（例: 標準ケースで月18.5万）。これは「貯めすぎの人に"使っていい"を返す」目的とは整合。文言は「〜まで（上限）」として天井であることを明示し、退職後の自由支出額を補足に添える。

### 5.7 v1 で省くもの
AI相談（§7）・Notion保存（§6）は中継が要るので後フェーズ。まず **ホーム入力 →（詳細でオプション）→ 計算 → 結果（3系列グラフ＋退職後）→ 振り返り＋提案** を端末内で完結（スターターUIをこの構成に置き換える）。提案文はv1ルールベース、後でAI相談に接続。

## 7. AI相談層（→ ADR 0615-ai-advisory）

- **相棒型**（断定する占い師ではない）。ユーザーが挙げた選択肢を**シミュレーションの入力に変換して結果を見せる**（回答の主役は言葉でなく未来の差分）
- **自己モデル接続**（欲求・制約・蓄積された主観反応を参照）で非汎用化
- **断定的投資助言を避ける**（特定銘柄推奨・「儲かる」断定はしない）→ 金商法グレー回避
- 「もっと稼ぐ」だけでなく「**もう足りている**」も返せる
- 実装: Anthropic API 直叩き（薄いラッパ1枚まで）。中継経由でキーを隠す

## 8. 前提・割り切り（実装時に効く）

- **平均一本線**: 「自分にだけ大きな不運」（テールリスク）は構造上出ない。本人合意済み
- **現在価値**: 物価を扱わない。昇給率・運用利回りは実質。将来額も「今の円」
- **合成コホート**: 年齢別の賃金・消費・資産は1時点の横断データで、同一個人の生涯推移ではない（世代効果・生存バイアス）。提示時にも「確定予測ではない」旨を添える
- **病気＝ベース消費に内包**（平均医療費）。大病は別枠の一回スポット
- **転職＝無視**、**暴落＝投資の平均利回りに内包**
- **イベントは終了年なし**（長期で過大＝悲観側）
- 持ち家は住居費が低く出る／住民税は1年遅れ 等の統計の癖あり

## 9. 確定した論点（2026-06-19 ユーザー確定）＋ 残課題

| 論点 | 確定 |
|---|---|
| 1. 毎月の支出の定義 | **総支出**（固定費＋生活費＋今かかっているオプション費用すべて） |
| 2. 将来消費のベース | **年齢カーブで増減**（消費増加率・中年ピーク→退職後減。実装ADR追補 決定F「消費一定」を**改訂**）＋ オプション3類型（状態変化/特定上乗せ/独立）で二重計上回避。物価は別途「無視（現在価値）」で確定 |
| 3. 退職後の収入 | **年金（収入転換）＋ 任意の「退職後パート」**（§3.6） |
| 4. 計算の終端 | **寿命まで**（§3.1 の寿命入力。退職後の取り崩しと「資産が尽きる年」を描く） |

**残る未決（数値は実装で確定。要導出は年齢昇給率・消費増加率とも解消済み・§3.2/§3.3）**:
- 年金額: **導出済み**（§3.7。国民約5.9万/厚生約15.1万・月、種類別）。退職後最低生活費≈14.9万/月。退職後パートの既定年額は手入力（例 年100万）
- what-if 計算の置き場所（端末内推奨・即応／重ければ中継）
